When thinking of this project I didn't know what I wanted to do. I asked myself "what can showcase my progess so far in learning html, css, and javascript?"

As of the making of this project, I recently began the Google UX Certification course. I wanted to do this certification to both boost and bolster my schooling. I wanted to be able to not only learn the process and thoughts behind UX design so in my classes I can focus more on the designing, not on why the designing works, but also I wanted to learn how to conduct the whole process of UX design so I can begin doing freelance work.

During my schooling I have realized that there are many different ways to use the tools within the Adobe suite, and some of those uses are often overlooked. Because of this, knowing that I would be learning Figma both in the certification course and in my upcoming classes I wanted to do a deep dive into Figma's main tools. I want to be able to learn not only the basic purposes of the main tools used, but I also wanted to be able to learn some of the hidden or not as commonly used uses of these tools. My intent in doing so is to be able to make sure I use Figma to its fullest potential so I can make the best designs possible with it.

What is Figma?

Figma began being developed by Dylan Field and Evan Wallace in 2012 with the original object to enable "anyone to be creative by creating free, simple creative tools in a browser."1 Figma was first released to the public on September 27, 2016.

Today Figma "is a powerful, collaborative design tool for teams."2 Its purpose is to enable teams to work on a document together in real time. It allows for quick, easy prototyping, even allowing branching off to try ideas without impacting the final design. When looking for a program that offers versatile UX/UI creation, Figma is an option that is hard to beat.
